# 离合词发现研究报告

## 课程总结

本课程的核心思想是「三要素」（🟦单元 / 🟧关系 / 🟨属性）在不同系统中的普适性。
通过期末大作业，实现了从理论到实践的完整闭环：

1. **句子结构**：用 GPF 将生语料解析为词序列和词性标注（智能体①）。
2. **语料库检索**：用 BCC 建倒排索引、执行 A B / A*B 模式查询（智能体②）。
3. **统计分析 + 大模型分析**：解析命中结果生成 JSONL，调用 DeepSeek V4 分析插入成分的语法类型和离合模式（智能体③）。
4. **知识库查询**：用 JSS 在 JSONL 上建多类型索引、执行 SQL 分析（智能体④）。

> 学懂这门课 = 学会用同一双眼睛看 自然语言 / 程序 / 语料 / 知识 / 智能体。

## 一、研究目标

利用 LangSC 工具包（GPF/BCC/JSS）和 LangGraph 智能体流水线，
从中文语料中自动发现并统计分析「离合词」现象。
离合词指汉语中可拆开使用的双音节词，如「见面」→「见过两次面」。

## 二、方法概述

### 2.1 四智能体流水线架构

```
corpus/corpus.txt（生语料）
    │
    ▼  ① 结构分析智能体（GPF）— 分词 + 词性标注
   segment/seg.txt
    │
    ▼  ② BCC 检索智能体 — 建索引 + 模式查询
   hits（命中上下文）
    │
    ▼  ③ 统计分析智能体 — 聚合 → JSONL + cfg + LLM 分析（DeepSeek V4）
   stats/liheci.jsonl + stats/cfg_liheci.txt + stats/llm_analysis.json
    │
    ▼  ④ JSS 检索智能体 — 自动建索引 + SQL
   answers（SQL 查询结果）
```

### 2.2 BCC 检索策略

| 模式 | 语法 | 说明 |
|------|------|------|
| 合用（紧邻） | `A B`（空格隔开） | A 和 B 紧邻出现 |
| 离用（同句） | `A*B`（* 通配符） | A 和 B 同句、任意距离 |

### 2.3 三要素视角

| 系统 | 🟦 单元 | 🟧 关系 | 🟨 属性 |
|------|---------|---------|---------|
| 智能体流水线 | 4 个 Node | 3 条 Edge | State 共享字段 |
| 句子结构 (GPF) | 词 Token | 依存关系 / 线性序 | POS 词性 |
| 语料库 (BCC) | 词条/句子 | 共现/位置 | 频次/上下文 |
| 知识库 (JSS) | JSONL 记录 | SQL 关系 | 索引类型 |

## 四、JSS SQL 查询结果

共执行 **3** 条 SQL 查询：

### SQL: `SELECT word, distance FROM liheci WHERE distance >= 1 ORDER BY word`
返回 **4** 条结果

```json
{"distance": 1, "word": "吃饭"}
{"distance": 2, "word": "吃饭"}
{"distance": 3, "word": "帮忙"}
{"distance": 3, "word": "生气"}
```

### SQL: `SELECT word, distance FROM liheci WHERE distance >= 2 ORDER BY distance DESC`
返回 **3** 条结果

```json
{"distance": 3, "word": "生气"}
{"distance": 3, "word": "帮忙"}
{"distance": 2, "word": "吃饭"}
```

### SQL: `SELECT word, distance FROM liheci WHERE distance >= 0 ORDER BY distance DESC`

```json
... 共 50 条
```

### SQL 说明

| 序号 | 索引类型 | 说明 |
|------|---------|------|
| 1 | number 范围 | 查询插入距离 >= 1 的离用记录（ORDER BY word） |
| 2 | number 范围 | 查询插入距离 >= 2 的高离用实例（ORDER BY distance DESC） |
| 3 | number 扫描 | 查询所有记录按距离排序（ORDER BY distance DESC） |

> 注：JSS C 扩展当前版本对 KV 索引过滤、GROUP BY 聚合和 LIKE 全文检索支持有限，
> 上述聚合统计改用 Python 侧计算作为补充。

## 五、大模型（LLM）分析结果

分析方法：**llm**（DeepSeek V4）

### 5.1 整体发现

离合词离用形式多样，插入成分包括定语、补语、数量结构和疑问词等，体现了汉语的灵活性。

### 5.2 插入成分类型分布

| 类别 | 出现次数 |
|------|---------|
| 数量结构 | 1 |
| 定语修饰 | 1 |
| 补语 | 1 |
| 疑问否定 | 1 |
| 动态助词 | 0 |
| 其他 | 0 |

### 5.3 识别到的离合模式

- V+定语+N
- V+补语+N
- V+疑问词+N
- V+数量+N

### 5.4 分词分析

| 离合词 | 插入成分 | 分析 |
|--------|---------|------|
| 吃饭 | 完、什么 | 可插入补语'完'或疑问词'什么'，分别形成'吃+补语+饭'和'吃+疑问词+饭'结构。 |
| 帮忙 | 小英的 | 插入定语成分，形成'帮+定语+忙'结构。 |
| 生气 | 一肚子 | 插入数量结构'一肚子'，形成'生+数量+气'结构。 |
